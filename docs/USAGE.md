

# Dois Compiler (doisc) — Usage Guide

## Overview

`doisc` is a compiler that takes `.dois` source files and transpiles them into C.

Current pipeline:


`.dois` source file
-> parse
-> type check
-> generate C (out.c)


The generated C depends on a small runtime located in `src/codegen/runtime/`.

---

## Building the Compiler

From the project root:

```bash
shards build
```

This produces:

```
./bin/doisc
```

---

## Running the Compiler

### Dev mode (via shards)

```bash
shards run doisc -- path/to/file.dois
```

### Using the compiled binary

```bash
./bin/doisc path/to/file.dois
```

---

## Output

The compiler always generates:

```
./out.c
```

This file contains:

- `#include "runtime/runtime.h"`
- all transpiled declarations (functions, types)
- a generated `dois_main()` function
- a generated `main()` wrapper

---

## Compiling the Generated C

To build a native binary:

```bash
cc out.c src/codegen/runtime/runtime.c -o out
```

Then run:

```bash
./out
```

---

## Generated C Structure

A typical generated file looks like:

```c
#include "runtime/runtime.h"

int64_t foo() {
  return 5;
}

void dois_main(void) {
  // top-level Dois code
}

int main(void) {
  dois_runtime_init();
  dois_main();
  dois_runtime_shutdown();
  return 0;
}
```

---

## Runtime

The runtime provides:

- `DoisResult`
- `DoisArray`
- `DoisMap`
- basic helpers (init/shutdown, constructors)

Files:

```
src/codegen/runtime/runtime.h
src/codegen/runtime/runtime.c
```

---

## Development Workflow

Typical loop:

```bash
shards run doisc -- file.dois
cat out.c
cc out.c src/codegen/runtime/runtime.c -o out
./out
```

---

## Current Limitations

- Top-level AST still originates from a single procedure
- Declaration vs execution splitting is in progress
- Not all expressions/statements are fully lowered
- No module/import system yet
- No automatic native compilation (manual `cc` step required)

---

## Notes

- Runtime symbols are prefixed with `dois_`
- User code is lowered into `dois_main()`
- The C `main()` function is always generated by the compiler
